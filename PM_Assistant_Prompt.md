# 🎯 产品经理 AI 协作助手

> 基于实际协作经验优化的提示词工具，帮助产品经理高效完成日常工作

---

## 📌 快速指令

复制以下指令直接使用：

### 1️⃣ 竞品截图采集

```
我需要下载 [App名称] 的产品截图来分析流程。
网站地址：[URL]
重点关注模块：[Onboarding/支付/核心功能等]
需要登录会员，请使用我已登录的Chrome窗口。
```

### 2️⃣ 截图分类与命名

```
帮我把下载的截图按产品流程逻辑分类：
1. 使用扁平文件夹结构（方便导入Figma）
2. 命名格式：[阶段编号]_[阶段名]_[步骤编号]_[页面描述].png
3. 重点模块 [模块名] 需要细颗粒度分类
4. 生成截图清单CSV
```

### 3️⃣ 流程分析报告

```
基于这些截图，帮我分析 [App名称] 的 [模块名] 流程：
1. 流程步骤拆解
2. 设计亮点总结
3. 可借鉴的点
4. 输出Markdown文档
```

### 4️⃣ 竞品对比分析

```
我有以下竞品的截图：
- [App1]：[文件夹路径]
- [App2]：[文件夹路径]

请对比分析它们的 [Onboarding/支付/XX功能] 流程：
1. 流程步骤对比表
2. 各自优缺点
3. 最佳实践建议
```

### 5️⃣ PRD 辅助生成

```
我要设计一个 [功能名称]，参考竞品 [App名称] 的 [模块]。
请帮我：
1. 梳理用户故事
2. 列出功能点清单
3. 生成流程图描述
4. 输出PRD框架
```

### 6️⃣ 设计规范提取

```
分析这些截图，提取 [App名称] 的设计规范：
1. 颜色系统
2. 字体层级
3. 间距规律
4. 组件样式
5. 交互模式
```

## 🔧 工作流模板

### A. 竞品研究完整流程

```
【项目】[项目名称] 竞品研究
【目标】分析 [竞品列表] 的 [模块] 设计

请按以下步骤执行：

Step 1: 截图采集
- 从 screensdesign.com / mobbin.com 下载截图
- 需要使用我已登录的会员账户

Step 2: 截图整理
- 按产品流程分类
- 扁平结构，统一命名
- 生成截图清单

Step 3: 流程分析
- 拆解每个流程步骤
- 标注设计亮点
- 生成分析文档

Step 4: 输出交付物
- 分类后的截图文件夹
- 流程分析Markdown
- 截图清单CSV

所有文件整合到一个项目文件夹中。
```

### B. 新功能设计参考

```
【功能】[功能名称]
【参考】[竞品App列表]

请帮我：
1. 收集参考竞品的相关截图
2. 分析各竞品的实现方式
3. 总结最佳实践
4. 给出设计建议
5. 生成功能点清单

输出格式：Markdown + CSV
```

### C. Onboarding 专项分析

```
分析 [App名称] 的 Onboarding 流程：

请细化到每一步：
1. 页面类型（信息收集/激励/结果展示）
2. 收集的信息点
3. 文案策略
4. 交互设计
5. 转化漏斗位置

输出：
- 流程步骤表格
- 设计亮点总结
- 可复用模式
```

## 🛠️ 技术配置

### Chrome 调试模式启动命令（Windows）

```powershell
# 关闭所有Chrome后执行
& "C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir=C:\ChromeDebug
```

### Python 依赖

```
selenium>=4.0.0
Pillow>=9.0.0
requests>=2.28.0
```

### 截图标准规格

- 宽度：402px（高度按比例）
- 格式：PNG
- 命名：英文+下划线，无空格

## 💡 协作技巧

### ✅ 高效沟通

1. **明确目标** - 说清楚最终要什么交付物
2. **指定格式** - 文件结构、命名规则、文档格式
3. **分步确认** - 复杂任务分步执行，每步确认
4. **给出约束** - 如"方便导入Figma"这样的使用场景

### ✅ 常用约束词

- "扁平结构" - 不要嵌套文件夹
- "细颗粒度" - 每个步骤单独分类
- "按流程顺序" - 文件名排序即流程顺序
- "可导入Figma" - 简单结构+清晰命名
- "生成CSV" - 方便Excel分析

### ✅ 批量操作

- 一次说清所有需求，我会自动规划步骤
- 可以说"全部执行"让我连续完成
- 中途可以调整，我会适应
